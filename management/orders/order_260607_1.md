# Management Order: Decouple Management Order Gating from Local Tests

## Direction

Staging plays in the playhouses must be an act of creative freedom, independent of top-down system infrastructure gating. We are delegating all plan/order checklist verification to the engine and removing staging-specific orders from the playhouses. While the Grimm house has no legacy staging orders to clean up, we must still update its tests to scan the root top-down.

## Orders

1. The Theatre Manager (hunt) shall update `@chbrain/khai-tests` and `@chbrain/khai-arch` to the latest version.
2. The Theatre Manager (hunt) shall update the project validation call in `tests/house.test.mjs` to scan the project root instead of limiting it to `plays/`.

## Implementation

- Update dependencies in `package.json`.
- Modify `tests/house.test.mjs` to remove the restricted `contentDir` argument.
- Run `npm test` to verify conformance.

## Targets

- [x] Update project validation to scan root in `tests/house.test.mjs`
- [x] Update dependencies in `package.json`
- [x] Verify local tests pass (`npm test`)
